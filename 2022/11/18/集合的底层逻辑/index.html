1. List接口：

1.1 ArrayList类

ArrayList维护了一个Object类型的数组：Object[] elementData

创建对象时如果使用的无参构造，则数组初始容量为0，加第一个元素后容量变为10。

为方便描述和区分扩容和临界，接下来【数组的容量】指数组的整体长度，【数组的长度】指数组中已经被存放元素的个数(不包含数组初始化时赋值的null)

如果使用指定大小(int)的有参构造，则数组初始化为指定大小；如果有参构造的参数是一个集合，会初始化成一个该传入集合的复制数组。

扩容机制为原来容量的1.5倍，在数组长度到达容量时（也就是容量的1.0倍）会促发扩容机制，称1.0为临界因子

比如无参构造创建后的数组容量为10，长度到达10后，容量扩容为15

1.2 Vector类

Vector也是维护了一个Object类型的数组：Object[] elementData，和ArrayList类似。但相较于ArrayList，Vector效率不高，但Vector的方法都有synchronized修饰，Vector的操作是线程安全的。

Vector的无参构造直接就将数组初始化为10。

扩容机制为原来容量的2倍，临界因子为1.0。

tips：Vector的操作方法虽然有Synchronized修饰，但只能说是相对线程安全的。

只能避免多个线程同时操作一个方法，多个线程操作多个方法的时候还是会出问题，这时就需要使用同步代码块解决了

1.3 LinkedList类

LinkedList实现了List接口和Deque接口，底层是一个双向的Node链表，因此可以在任意位置插入或删除元素。

小总结：

由于ArrayList底层是一个数组，所以在查找方面的时间复杂度是O(1)，而增删方面是O(N);

而LinkedList底层是双向链表，反而增删很容易，查询比较难

所以如果我们改查操作多，可以使用ArrayList或Vector；增删操作多选择LinkedList

底层结构 增删效率 改查效率
ArrayList 可扩容的数组 较低，通过数组扩容 较高
LinkedList 双向链表 较高，通过链表节点的追加或插入 较低

2.Set接口

2.1 HashSet类

(不可重复，在比较对象时需要重写equals方法)

HashSet是通过一个HashMap实现的，而HashMap的本质是个【数组 / 红黑树 + 链表】，所以存放元素时是无序的。

对HashMap的结构理解如下：

  1. HashMap的本质是一个数组，这个数组的类型是单向键值对Node，即：Node<K,V>[ ] table

  2. Node是一个有着next指针的节点，因此table数组的元素是一个Node链表

  3. table的索引确认是通过一个哈希函数 hash(key)实现的：

    return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);

数组初始容量是16，长度到达容量的0.75倍（临界因子）也就是 12 后 就会扩容成2倍（32）；

而数组的元素：【链表】的长度到达 8 后会开启判断：如果此时数组长度 ≥ 64，会将链表转化为红黑树，如果长度不足64，则扩容数组，重新分配所有元素链表的数组索引。

另外：毕竟是靠HashMap实现的，所以元素是以键：值的形式添加，k为要添加的元素，v为一个空Object【PRESENT】

【PRESENT】在HashMap是被static final修饰的，所以只有一份内存，每个v都是指向这个空对象

LinkedHashSet（有序）本质是个有顺序的双向链表数组，为了实现顺序性对除next之外每个节点增加before和after两个指针，这两个指针用于实现元素存放的顺序，next用于实现链表的节点连接

3.HashMap(无序)

HashMap本质也是个链表数组，数组长度初始值为16。但在此之上增加了一个EntrySet(存放table数组的引用)，keySet(Set)存放所有键，values(Collection)存放所有值

4.HashTable

HashMap本质也是个链表数组，数组长度初始值为11，临界因子为0.75。扩容：2x+1

key和value不允许为null，同步线程安全

但在此之上增加了一个EntrySet(存放table数组的引用)，keySet(Set)存放所有键，values(Collection)存放所有值。
